## What is a Workflow Execution?

A **durable, reliable, and scalable function execution** - the main unit of execution in a Temporal Application.

### Key Properties

| Property | Definition |
|----------|------------|
| **Durability** | No imposed time limit - runs seconds to years |
| **Reliability** | Fully recoverable after failure |
| **Scalability** | Millions to billions executing concurrently |

## Replay

Method by which Workflow Execution resumes progress:
- Commands generated during replay are checked against Event History
- Workflow picks up from last recorded event on failure

## Commands and Awaitables

### What a Workflow Execution Does
1. **Issue Commands** (requested actions)
2. **Wait on Awaitables** (Futures)

### Commands
Actions issued by Worker to Temporal Service after Workflow Task Execution completes:
- Recorded as Events in Event History
- Generated by Workflow APIs in your code
- Batched and sent when Workflow can't progress without Awaitable result

### Awaitables (Blocking Points)
Progress can block on:
- Explicit "Await" APIs
- Cancellation confirmation (other Workflow)
- Signal sent confirmation
- Child Workflow start/result
- Activity Execution result
- Timer firing

## Workflow Execution Status

### Open
- **Running**: Only Open status - actively progressing or waiting

### Closed
| Status | Meaning |
|--------|---------|
| **Cancelled** | Successfully handled cancellation request |
| **Completed** | Finished successfully |
| **Continued-As-New** | Continued as new execution |
| **Failed** | Returned error and failed |
| **Terminated** | Externally terminated |
| **Timed Out** | Reached timeout limit |

## Workflow Execution Chain

Sequence of Workflow Executions sharing same Workflow Id:
- Each link = **Workflow Run**
- Connected by:
  - Continue-As-New
  - Retries
  - Temporal Cron Job

### Identification
- Namespace + Workflow Id + Run Id = Unique identification

### Timeouts
- **Execution Timeout**: Applies to Chain
- **Run Timeout**: Applies to single Run

## Memo

Non-indexed metadata set at start time or in Workflow code:
- Returned on describe/list operations
- For organization and management
- **Not for critical execution data** (lacks type safety, eventual consistency)

## State Transitions

Unit of progress recorded in persistence store:
- Activity Heartbeat: 1-2 State Transitions
- Simple Workflow with 2 Activities: 11 State Transitions
  - 2 for Workflow start
  - 4 for each Activity
  - 1 for completion

---

## Takeaways

1. **One Workflow Execution = One durable function** - executes once to completion
2. **Replay ensures reliability** - state recreated from Event History
3. **Commands map to Events** - must match on replay
4. **Workflow can only block on SDK-provided Awaitables**
5. **State Transitions have cost** - minimize for large workloads
