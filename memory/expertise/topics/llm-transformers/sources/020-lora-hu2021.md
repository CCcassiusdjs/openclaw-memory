## 📖 Summary

LoRA (Low-Rank Adaptation) is a parameter-efficient fine-tuning method that freezes pre-trained model weights and injects trainable rank decomposition matrices into each Transformer layer. This dramatically reduces the number of trainable parameters while maintaining fine-tuning quality.

### Core Contribution
- **10,000x fewer trainable parameters** than full fine-tuning
- **3x lower GPU memory** requirement
- **No additional inference latency** (unlike adapters)
- **On-par or better performance** than full fine-tuning

---

## 🔑 Key Concepts Learned

### 1. Problem: Full Fine-Tuning is Expensive

GPT-3 175B requires:
- 175B parameters to update
- Massive GPU memory for gradients
- Separate model copy for each task

### 2. LoRA Insight: Low-Rank Adaptation

**Hypothesis:** Model adaptation has low "intrinsic dimension"

Instead of updating weights W₀ directly:
```
W = W₀ + ΔW
ΔW = B × A  (low-rank decomposition)
```

Where:
- B ∈ R^(d×r) - Learnable
- A ∈ R^(r×k) - Learnable
- r << min(d, k) - Rank (typically 4-64)

### 3. Rank Decomposition

```
Full update: ΔW requires d × k parameters
LoRA update: ΔW requires d × r + r × k parameters

Example (d=4096, k=4096, r=8):
  Full: 4096 × 4096 = 16,777,216 parameters
  LoRA: 4096 × 8 + 8 × 4096 = 65,536 parameters
  
Reduction: 256x fewer parameters
```

---

## 📊 Technical Details

### Architecture

```
Original: y = W₀x + bx
LoRA:     y = W₀x + BAx + bx

Where:
  W₀ ∈ R^(d_out × d_in) - Frozen
  B ∈ R^(d_out × r) - Trainable
  A ∈ R^(r × d_in) - Trainable
  r << d_in, d_out
```

### Initialization

- **A:** Random initialization (Gaussian)
- **B:** Zero initialization (so BA starts as zero)
- This ensures training starts from pre-trained model

### Where to Apply LoRA

LoRA can be applied to any weight matrix, but most effective on:
- **Attention weights** (Q, K, V, O)
- **Feed-forward weights** (less common)

Standard practice: Apply to Q and V projections only.

## 🔬 Key Insights

### 1. Intrinsic Dimension Hypothesis
Fine-tuning operates in a low-dimensional subspace. The "intrinsic dimension" needed for adaptation is much smaller than model size.

### 2. Rank Selection
- **r=4-8:** Sufficient for most tasks
- **r=16-64:** For complex tasks or large datasets
- **r > d:** Approaches full fine-tuning

### 3. No Inference Overhead
Unlike adapter layers:
- LoRA weights can be merged: W_new = W₀ + BA
- Single matrix multiplication at inference
- No additional latency

### 4. Task Switching
- Store only small LoRA weights per task
- Switch tasks by swapping LoRA weights
- Base model remains frozen

## 📈 QLoRA Extension (2023)

**Quantized LoRA (QLoRA):**
1. Quantize base model to 4-bit
2. Apply LoRA on top
3. Further reduces memory requirements

**Benefits:**
- Fine-tune 65B model on single 24GB GPU
- 16-bit LoRA computations on 4-bit base

## 🎓 Practical Implementation

### LoRA Hyperparameters

| Parameter | Typical Values | Notes |
|-----------|-----------------|-------|
| rank (r) | 4, 8, 16, 32, 64 | Higher for complex tasks |
| alpha (α) | r, 2r, 16 | Scaling factor |
| dropout | 0.05, 0.1 | Regularization |
| target_modules | ["q_proj", "v_proj"] | Where to apply LoRA |

### Scaling Rule
```
Output = W₀x + (α/r) × BAx

α/r acts as learning rate scaling
Common: α = r (so α/r = 1)
```
