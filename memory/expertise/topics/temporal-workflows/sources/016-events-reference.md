# Temporal Events Reference

**Source:** https://docs.temporal.io/references/events
**Date:** 2026-03-12
**Status:** read

---

## What is an Event?

Events are created by the Temporal Service in response to:
- External occurrences
- Commands generated by a Workflow Execution

All Events appear in a Workflow Execution's Event History.

---

## Workflow Lifecycle Events

### WorkflowExecutionStarted
- **Always the first Event** in Event History
- Indicates Temporal Service received request to spawn Workflow
- Contains: workflow_type, task_queue, input, timeouts, retry_policy, etc.

### WorkflowExecutionCompleted
- Workflow successfully completed
- Contains: result
- One of the last Events in History

### WorkflowExecutionFailed
- Workflow failed with error
- Contains: failure, retry_state

### WorkflowExecutionTimedOut
- Workflow timed out by Temporal Server
- Contains: retry_state, new_execution_run_id

### WorkflowExecutionCancelRequested
- Request made to cancel Workflow
- Contains: cause, identity

### WorkflowExecutionCanceled
- Client confirmed cancellation
- Contains: details

### WorkflowExecutionTerminated
- Workflow forcefully terminated
- Contains: reason, identity

### WorkflowExecutionContinuedAsNew
- Workflow completed and new Workflow started in same transaction
- Contains: new_execution_run_id, workflow_type, input, etc.

---

## Workflow Task Events

### WorkflowTaskScheduled
- Workflow Task scheduled
- Worker can process new history events

### WorkflowTaskStarted
- Worker picked up Workflow Task
- Processing new history events

### WorkflowTaskCompleted
- Workflow Task completed successfully
- May trigger additional Commands (Activities, Timers, etc.)

### WorkflowTaskTimedOut
- Workflow Task timed out
- Either no Worker available or processing took too long

### WorkflowTaskFailed
- Workflow Task failed
- Usually means non-deterministic Workflow

---

## Activity Events

### ActivityTaskScheduled
- Activity Task scheduled
- Contains: activity_id, activity_type, task_queue, input, timeouts, retry_policy

### ActivityTaskStarted
- Activity Task started by Worker
- Note: Not written to History until terminal Event occurs

### ActivityTaskCompleted
- Activity Task completed successfully
- Contains: result

### ActivityTaskFailed
- Activity Task failed
- Contains: failure, retry_state

### ActivityTaskTimedOut
- Activity timed out
- Contains: failure, timeout_type

### ActivityTaskCancelRequested
- Request to cancel Activity
- Contains: scheduled_event_id

### ActivityTaskCanceled
- Activity confirmed cancellation
- Contains: details, latest_cancel_requested_event_id

---

## Timer Events

### TimerStarted
- Timer started
- Contains: timer_id, start_to_fire_timeout

### TimerFired
- Timer fired
- Contains: timer_id, started_event_id

### TimerCanceled
- Timer canceled
- Contains: timer_id, started_event_id

---

## External Workflow Events

### RequestCancelExternalWorkflowExecutionInitiated
- Workflow requested to cancel another Workflow
- Contains: namespace, workflow_execution

### ExternalWorkflowExecutionCancelRequested
- Temporal Service requested cancellation of target Workflow

---

## Key Takeaways

1. **Events are immutable** - Once recorded, cannot be changed
2. **Event History = source of truth** - Used for replay
3. **Events follow Commands** - Each Command produces Events
4. **WorkflowTask events are meta-level** - They control Workflow execution
5. **Activity events track full lifecycle** - Scheduled → Started → Completed/Failed/TimedOut
