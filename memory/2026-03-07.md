## Infraestrutura de Teste Discutida

### Arquitetura Completa
```
Dataset Orchestrator → Sensor Emulator → MultiRad Framework → ArduCopter EKF
        ↓                      ↓                    ↓                    ↓
     CSV Files           LD_PRELOAD           fork/exec           Sistema sob teste
```

### Fluxo de Execução
1. **Fase 1: Setup** - Carregar params, copiar CSVs
2. **Fase 2: Radiação** - Injetar bit flip
3. **Fase 3: Execução** - fork() + LD_PRELOAD + exec()
4. **Fase 4: Coleta** - Parse log, extrair flags
5. **Fase 5: Classificação** - Classificar tipo de falha

### Classificação de Falhas
- **Type A (52%)** - Crash imediato, nenhum sensor completou
- **Type B (14%)** - GPS falhou, IMU/BARO/MAG OK
- **Type C (14%)** - Atitude falhou, GPS OK
- **Type D (7%)** - MAG falhou no final, GPS/IMU/BARO OK
- **Type E (7%)** - Parcial, GPS/BARO OK

### Output Vector
```c
typedef struct {
    uint32_t ekf_flags;          // Status bitmask
    uint32_t armed_successfully; // 1 se armou
    uint32_t total_gps_drops;    // GPS dropouts
    float pos_error_ned_m[3];    // Sensor status flags
    float att_error_rpy_deg[3];  // Warning/error counts
} output_vector_type;
```

### Flags Importantes
- Bit 31: RunFailed (0x80000000)
- Bit 10: ArmedSeen
- Bit 7: MagReplayOk
- Bit 6: BaroReplayOk
- Bit 5: ImuReplayOk
- Bit 4: GpsReplayOk
- Bit 0: ProcessOk
