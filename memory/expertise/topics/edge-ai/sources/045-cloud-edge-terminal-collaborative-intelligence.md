# Cloud-Edge-Terminal Collaborative Intelligence in AIoT (2025)

**Source:** https://arxiv.org/html/2508.18803v1
**Type:** Academic Survey (arXiv)
**Date:** August 2025
**Relevance:** ⭐⭐⭐⭐⭐

## Summary

Comprehensive survey on Cloud-Edge-Terminal Collaborative Intelligence (CETCI) for AIoT networks. Covers architectural components, collaboration paradigms, enabling technologies, and applications across manufacturing, transportation, healthcare, and agriculture.

## Key Concepts

### Three-Tier Architecture

| Layer | Components | Role |
|-------|------------|------|
| **Cloud** | Central data centers, powerful compute | Global insights, large-scale training |
| **Edge** | MEC nodes, gateways, micro data centers | Reduced latency, localized processing |
| **Terminal** | IoT sensors, actuators, mobile devices | Data collection, preliminary analysis |

### Motivation for Collaboration

**Terminal Device Limitations:**
- Limited processing power
- Memory constraints
- Energy constraints
- Duty-cycled connectivity

**Network Challenges:**
- Bandwidth bottlenecks
- Real-time demands (autonomous vehicles, health monitoring)
- Scalability (millions of devices)
- Privacy concerns (sensitive data)

### Collaboration Paradigms

1. **Task Offloading** - Where to execute AI tasks
2. **Resource Allocation** - How to distribute compute/memory/network
3. **Model Partition** - How to split models across tiers
4. **Knowledge Transfer** - How to share learned representations

### Core Enabling Technologies

**Network Technologies:**
- Network Virtualization (NFV)
- Software-Defined Networking (SDN)
- Container Orchestration (Kubernetes, Docker)

**Computing Paradigms:**
- Edge Computing
- Fog Computing
- Mobile Edge Computing (MEC)
- Cloud-Edge Continuum

### Intelligent Collaboration Frameworks

| Framework | Description |
