## 🔑 Conceitos-Chave

### Definição
> "A digital twin is a virtual representation of a physical object or system that uses real-time data to accurately reflect its real-world counterpart's behavior, performance and conditions."

### Características Principais
1. **Real-time, two-way data exchange** - Comunicação bidirecional
2. **Continuous monitoring** - Monitoramento contínuo
3. **Simulation capability** - Capacidade de simulação
4. **Lifecycle coverage** - Design → Production → Maintenance → Decommissioning

| Componente | Função |
|------------|--------|
| **Physical Asset** | Objeto físico a ser monitorado |

### 4. Analysis & Simulation

## 📊 Tipos de Digital Twins

### Hierarquia de 4 Níveis

1. **Component Twins** (Part Twins)
   - Componentes individuais
   - Ex: válvula, motor, turbina
   - Maior granularidade

2. **Asset Twins**
   - Unidades funcionais completas
   - Múltiplos componentes interagindo
   - Ex: sistema de válvulas, drive train

3. **System Twins**
   - Sistemas integrados
   - Múltiplos assets trabalhando juntos
   - Ex: segmento de pipeline, turbina completa

4. **Process Twins**
   - Visão mais ampla
   - Facility, supply chain, workflow completo
   - Ex: rede de distribuição, fábrica inteira

## 🆚 Comparações

### DT vs Simulation
| Aspecto | Digital Twin | Simulation |
|---------|-------------|------------|
| **Conexão** | Real-time com físico | Estático, isolado |
| **Dados** | Contínuos | Predefinidos |
| **Feedback** | Envia para físico | Não envia |

### DT vs 3D Model
- **3D Model**: Estático, snapshot no tempo
- **Digital Twin**: Dinâmico, real-time, comportamento

### DT vs Digital Thread
- **DT**: Otimizar assets e processos
- **Thread**: Visão organization-wide
