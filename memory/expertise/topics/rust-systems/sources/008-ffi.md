## Resumo Principal

FFI permite chamar código C de Rust e vice-versa. Usa `extern "C"` para declarar funções C, `#[link]` para linkar bibliotecas, e `unsafe` para chamar funções externas. Criar wrappers seguros é essencial.

---

## Conceitos-Chave

### 1. Declarando Funções Externas
```rust
use libc::size_t;

#[link(name = "snappy")]
unsafe extern "C" {
    fn snappy_max_compressed_length(source_length: size_t) -> size_t;
}

fn main() {
    let x = unsafe { snappy_max_compressed_length(100) };
    println!("max compressed length: {}", x);
}
```

**Componentes:**
- `#[link(name = "...")]` - Linka biblioteca nativa
- `extern "C"` - Especifica C ABI
- `unsafe` - Funções externas são unsafe

### 2. Tipos C em Rust
```rust
use libc::{c_int, size_t, c_char};

// Correspondência de tipos:
// C: int        → Rust: libc::c_int
// C: size_t     → Rust: libc::size_t
// C: char*      → Rust: *const c_char
// C: void*      → Rust: *mut c_void
// C: uint8_t*   → Rust: *mut u8
```

### 3. Criando Wrappers Seguros
```rust
// Função C unsafe
unsafe extern "C" {
    fn snappy_validate_compressed_buffer(src: *const u8, len: size_t) -> c_int;
}

// Wrapper seguro
pub fn validate_compressed_buffer(src: &[u8]) -> bool {
    unsafe {
        snappy_validate_compressed_buffer(src.as_ptr(), src.len() as size_t) == 0
    }
}
```

**Padrão:** unsafe internals, safe interface

### 4. Buffers de Saída
```rust
pub fn compress(src: &[u8]) -> Vec<u8> {
    unsafe {
        let srclen = src.len() as size_t;
        let psrc = src.as_ptr();
        
        // Aloca buffer com capacidade máxima
        let mut dstlen = snappy_max_compressed_length(srclen);
        let mut dst = Vec::with_capacity(dstlen as usize);
        let pdst = dst.as_mut_ptr();
        
        // Comprime
        snappy_compress(psrc, srclen, pdst, &mut dstlen);
        
        // Define tamanho real
        dst.set_len(dstlen as usize);
        dst
    }
}
```

**Key points:**
- `Vec::with_capacity()` aloca sem inicializar
- `as_ptr()` / `as_mut_ptr()` converte para raw pointers
- `set_len()` define tamanho após preencher

### 5. Chamando Rust de C
```rust
// Rust side
#[unsafe(no_mangle)]
pub extern "C" fn hello_from_rust() {
    println!("Hello from Rust!");
}
```

```c
// C side
extern void hello_from_rust();

int main(void) {
    hello_from_rust();
    return 0;
}
```

**Key points:**
- `#[no_mangle]` - Preserva nome do símbolo
- `extern "C"` - Usa C calling convention
- `crate-type = ["cdylib"]` - Compila como shared library

### 6. Callbacks de C para Rust
```rust
// Callback simples
extern fn callback(a: i32) {
    println!("Called from C with value {}", a);
}

#[link(name = "extlib")]
unsafe extern "C" {
    fn register_callback(cb: extern fn(i32)) -> i32;
    fn trigger_callback();
}

fn main() {
    unsafe {
        register_callback(callback);
        trigger_callback();  // C chama callback()
    }
}
```

### 7. Callbacks com Estado (Rust Objects)
```rust
struct RustObject {
    a: i32,
}

unsafe extern "C" fn callback(target: *mut RustObject, a: i32) {
    println!("Called with value {}", a);
    (*target).a = a;  // Atualiza objeto Rust
}

#[link(name = "extlib")]
unsafe extern "C" {
    fn register_callback(
        target: *mut RustObject,
        cb: unsafe extern fn(*mut RustObject, i32)
    ) -> i32;
}
```

### 8. Acessando Globais C
```rust
#[link(name = "readline")]
unsafe extern "C" {
    static rl_readline_version: libc::c_int;
}

fn main() {
    println!("Version: {}", unsafe { rl_readline_version as i32 });
}
```

### 9. Modificando Globais C
```rust
#[link(name = "readline")]
unsafe extern "C" {
    static mut rl_prompt: *const libc::c_char;
}

fn main() {
    let prompt = CString::new("prompt> ").unwrap();
    unsafe {
        rl_prompt = prompt.as_ptr();
    }
}
```

## Atributos Importantes

| Atributo | Uso | Descrição |
|----------|-----|-----------|
| `#[link(name = "foo")]` | extern block | Linka biblioteca nativa |
| `#[link(name = "foo", kind = "static")]` | extern block | Linka biblioteca estática |
| `#[link(name = "CoreFoundation", kind = "framework")]` | extern block | Linka framework (macOS) |
| `#[no_mangle]` | fn | Preserva nome do símbolo |
| `extern "C"` | fn/block | Usa C calling convention |

## Calling Conventions

| ABI | Uso |
|-----|-----|
| `"C"` | C calling convention (default) |
| `"stdcall"` | Windows API (x86) |
| `"fastcall"` | Passar args em registradores |
| `"system"` | ABI padrão do sistema |
| `"win64"` | Windows x86_64 |
| `"sysv64"` | Linux x86_64 |

## Padrões Importantes

### Pattern: Safe Wrapper
```rust
// Biblioteca C unsafe
unsafe extern "C" {
    fn raw_compress(src: *const u8, len: size_t, dst: *mut u8) -> c_int;
}

// Wrapper seguro
pub fn compress(data: &[u8]) -> Result<Vec<u8>, Error> {
    let mut output = Vec::with_capacity(data.len() * 2);
    unsafe {
        let result = raw_compress(
            data.as_ptr(),
            data.len(),
            output.as_mut_ptr()
        );
        if result < 0 {
            return Err(Error::CompressFailed);
        }
        output.set_len(result as usize);
    }
    Ok(output)
}
```

### Pattern: RAII com FFI
```rust
struct RawResource {
    ptr: *mut c_void,
}

impl Drop for RawResource {
    fn drop(&mut self) {
        unsafe {
            raw_free(self.ptr);
        }
    }
}
```

### Pattern: CString para C Strings
```rust
use std::ffi::CString;

let c_string = CString::new("hello").unwrap();
let ptr: *const c_char = c_string.as_ptr();
```

## Insights para Systems Programming

1. **Always Wrap Unsafe:** Nunca expor funções FFI diretamente
2. **Use libc Types:** `libc::c_int`, `libc::size_t`, etc. para compatibilidade
3. **Lifetime Management:** C não sabe sobre lifetimes - gerenciar manualmente
4. **Error Handling:** Converter c_int de retorno para Result<T, E>
5. **Memory Safety:** Vec::with_capacity + set_len para output buffers
6. **Callbacks:** Usar *mut T para passar estado para callbacks C

## Comparação com C/C++

| Feature | C/C++ FFI | Rust FFI |
|---------|-----------|----------|
| Linking | `-l library` | `#[link(name = "...")]` |
| ABI | Default C | Explicit `extern "C"` |
| Safety | Manual | Enforced by type system |
| Types | Implicit | Explicit `libc::` types |
| Strings | `char*` | `CString` / `CStr` |
